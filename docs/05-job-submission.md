Jobs can be submitted through Swagger using:

```text
POST /api/jobs/jdl
```

## Simple JDL example

Example body:

```json
[
  "Arguments = \"jobDescription.xml -o LogLevel=INFO\";\nExecutable = \"dirac-jobexec\";\nJobName = jobName_1;\nJobGroup = group_1;\nJobType = User;\nSite = ANY;"
]
```

This submits one job because the JSON array contains one JDL string.

## Submit multiple different jobs

To submit multiple jobs, provide multiple JDL strings:

```json
[
  "Arguments = \"jobDescription.xml -o LogLevel=INFO\";\nExecutable = \"dirac-jobexec\";\nJobName = jobName_1;\nJobGroup = group_1;\nJobType = User;\nSite = ANY;",

  "Arguments = \"jobDescription.xml -o LogLevel=DEBUG\";\nExecutable = \"dirac-jobexec\";\nJobName = jobName_2;\nJobGroup = group_2;\nJobType = User;\nSite = ANY;"
]
```

## JDL fields

Example:

```text
Arguments = "jobDescription.xml -o LogLevel=INFO";
Executable = "dirac-jobexec";
JobName = jobName_1;
JobGroup = group_1;
JobType = User;
Site = ANY;
```

Meaning:

| Field | Meaning |
|-------|---------|
| `Arguments` | Command-line arguments passed to the executable |
| `Executable` | Program executed by DIRAC |
| `JobName` | Human-readable job name |
| `JobGroup` | Group or category for organizing jobs |
| `JobType` | Type of job, usually `User` |
| `Site` | Where the job may run; `ANY` allows any compatible site |

## `jobDescription.xml`

The command:

```text
dirac-jobexec jobDescription.xml -o LogLevel=INFO
```

uses `jobDescription.xml` as an execution description. It is typically generated by DIRAC from the submitted job description and used by `dirac-jobexec` to execute the job payload.
